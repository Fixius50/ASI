Manifiesto - HubMaster SPAVisiónCrear una plataforma centralizada (Hub) que sirva como punto de entrada para múltiples sub-proyectos. La aplicación debe ser ligera, altamente compatible con Vercel y capaz de gestionar rutas dinámicas para mostrar contenido específico de cada sub-proyecto sin perder la persistencia de la SPA.Stack TecnológicoFrontend: HTML5, Tailwind CSS (Styling), Vanilla JavaScript (Router).Deployment: Vercel (Configuración de rewrites y headers).Arquitectura: SPA (Single Page Application) con soporte para sub-directorios estáticos.Reglas de DiseñoEstética: Dark mode por defecto, tipografía Sans-serif limpia (Inter/System).Layout: Sidebar o Grid de navegación para los proyectos.Componentes: Tarjetas interactivas para previsualizar sub-proyectos.Estrategia de ArchivosEl archivo index.html en la raíz manejará la lógica de navegación. Los proyectos hijos residirán en carpetas como /projects/proyecto-1/, manteniendo su propia independencia si es necesario, o siendo consumidos por la SPA principal.
